A Ordem dos Fatores Altera (e Muito) o Produto

Um computador não tem "bom senso". Ele executa as instruções na ordem exata em que as recebe. Se a ordem estiver errada, o resultado será, na melhor das hipóteses, incorreto e, na pior, um desastre.

Exemplo: Calçar Meias e Sapatos

Ordem Incorreta
  1. Pegar o sapato.
  2. Calçar o sapato.
  3. Pegar a meia.
  4. Calçar a meia. ⬅️ Impossível!
Ordem Correta
  1. Pegar a meia.
  2. Calçar a meia.
  3. Pegar o sapato.
  4. Calçar o sapato. ✔️ Sucesso!

Esta analogia simples ilustra um princípio fundamental: a sequência de um algoritmo é sagrada. Mudar a ordem de duas linhas de código pode mudar completamente o comportamento de um programa.

Atividade Prática: O Desafio do "Fishbowl"

Agora, vamos praticar a criação de algoritmos em grupo. A metodologia "Fishbowl" (aquário) funciona da seguinte forma:

  • Um pequeno grupo de alunos senta-se no centro da sala (o "aquário").
  • Este grupo recebe um problema e deve discuti-lo e criar um algoritmo no quadro para o resolver.
  • Os restantes alunos observam de fora, sem intervir.
  • Após um tempo, os grupos trocam. Os observadores entram no "aquário" para refinar, corrigir ou criar uma nova solução para o mesmo problema (ou um novo).

Problemas para a Atividade:

Problema A: Trocar uma Lâmpada

Crie um algoritmo detalhado para trocar uma lâmpada queimada num candeeiro de teto. Pense em segurança, ferramentas necessárias e possíveis problemas.

Problema B: Fazer Matrícula

Crie um algoritmo com os passos que um aluno deve seguir para se matricular numa nova disciplina da faculdade, desde a consulta de horários até à confirmação.
